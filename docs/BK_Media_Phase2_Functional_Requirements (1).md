## 1. Overview

Phase 2 adds equipment tracking and warehouse management to the Video Department CRM.

**Flow:**
```
Equipment master → Kit creation → Quotation approved
→ Warehouse check (available / vendor needed)
→ Operator assignment → Dispatch
```

**Key concepts:**
- **Equipment master** — complete inventory of all 223 items with purchase price
- **Kit** — camera body + fixed accessories as one unit
- **Warehouse check** — per-event availability check with vendor fallback
- **Vendor** — external rental vendors when BK Media stock is unavailable
- **Asset value** — total purchase price of all equipment for financial tracking

## 4. Module 3 — Warehouse Check

### 4.1 Warehouse Check Screen

**Trigger:** Quotation approved → warehouse check screen auto-opens for that inquiry.

**Screen sections:**
1. Event info strip (read-only): client, dates, quotation no.
2. Metrics: Total / Available / Vendor needed / Confirmed
3. Available section (green rows) — BK Media stock
4. Vendor needed section (amber rows) — external rental

### 4.2 Available section columns

| Column | Description |
|--------|-------------|
| Position | From quotation (Center Tally, Crane etc.) |
| Kit / Equipment | Matched kit or equipment |
| Serial no. | Serial number |
| Status | Available badge |
| Action | Confirm button |

### 4.3 Vendor needed section columns

| Column | Description |
|--------|-------------|
| Position | From quotation |
| Equipment | Type needed |
| Select vendor | Dropdown from vendor list |
| Cost/day (₹) | Editable |
| Total cost | cost × event days (auto) |
| Action | Save button |

### 4.4 Availability check logic

```javascript
// For each position in quotation:
// 1. Find matching equipment/kit
// 2. Check equipment_bookings for overlapping dates
// 3. If no overlap → AVAILABLE
// 4. If overlap → IN_USE (show which event)
// 5. If not owned → VENDOR_NEEDED
```

### 4.5 Booking logic

When confirmed:
- Creates record in `equipment_bookings`
- `booked_from` = inquiry start date, `booked_to` = inquiry end date
- On return: status → Available

## 6. Business Rules

| Rule | Description |
|------|-------------|
| Asset calculation | Sold/Retired items excluded from total |
| Purchase price | Optional but recommended for all items |
| Kit booking | All kit items booked together |
| Double booking | Same equipment cannot be booked for overlapping dates |
| Vendor cost | Internal only — not shown to client |
| Status auto-update | Equipment status auto "In use" on confirm, "Available" after event end |

## 8. Database Schema (Prisma)

```prisma
model Equipment {
  id             Int       @id @default(autoincrement())
  productName    String
  category       String
  // CAMERA | VIDEO_MIXER | VIDEO_RECORDER | AUDIO_MIXER
  // WIRELESS_TX | UPS | ACCESSORY
  quantity       Int       @default(1)
  serialNumber   String?
  bodyName       String?
  kitId          Int?
  kit            Kit?      @relation(fields: [kitId], references: [id])
  respPerson     String?
  purchaseDate   DateTime?
  purchaseFrom   String?
  billNumber     String?
  purchasePrice  Decimal?  @db.Decimal(12,2)   // Asset value ₹
  status         String    @default("AVAILABLE")
  // AVAILABLE | IN_USE | MAINTENANCE | SOLD | RETIRED
  notes          String?
  createdAt      DateTime  @default(now())
  updatedAt      DateTime  @updatedAt

  bookings       EquipmentBooking[]
}

model Kit {
  id          Int       @id @default(autoincrement())
  name        String    @unique
  description String?
  mainBodyId  Int?
  createdAt   DateTime  @default(now())

  items       Equipment[]
  bookings    EquipmentBooking[]
}

model EquipmentBooking {
  id               Int        @id @default(autoincrement())
  inquiryId        Int
  inquiry          Inquiry    @relation(fields: [inquiryId], references: [id])
  equipmentId      Int?
  equipment        Equipment? @relation(fields: [equipmentId], references: [id])
  kitId            Int?
  kit              Kit?       @relation(fields: [kitId], references: [id])
  position         String?
  bookedFrom       DateTime
  bookedTo         DateTime
  status           String     @default("BOOKED") // BOOKED | OUT | RETURNED
  vendorId         Int?
  vendor           Vendor?    @relation(fields: [vendorId], references: [id])
  vendorCostPerDay Decimal?   @db.Decimal(10,2)
  totalVendorCost  Decimal?   @db.Decimal(12,2)
  confirmedById    Int?
  confirmedAt      DateTime?
}

model Vendor {
  id             Int      @id @default(autoincrement())
  name           String
  phone          String
  email          String?
  specialization String?
  city           String?
  gstNumber      String?
  notes          String?
  isActive       Boolean  @default(true)
  createdAt      DateTime @default(now())

  bookings       EquipmentBooking[]
}
```

## 9. API Summary

### Equipment
```
GET    /equipment?category=&status=&search=&page=&limit=
POST   /equipment
GET    /equipment/:id
PUT    /equipment/:id
DELETE /equipment/:id         // soft delete → status = RETIRED
POST   /equipment/import-csv

GET    /equipment/asset-summary
       Returns: {
         totalValue: 5200000,
         activeCount: 216,
         byCategory: [
           { category: 'CAMERA', count: 10, value: 2400000 },
           { category: 'VIDEO_MIXER', count: 9, value: 800000 },
           ...
         ]
       }

GET    /equipment/availability?startDate=&endDate=
       Returns: [{ ...equipment, isAvailable, bookedInEvent? }]
```

### Kits
```
GET    /kits
POST   /kits
GET    /kits/:id
PUT    /kits/:id
DELETE /kits/:id
POST   /kits/:id/add-item        Body: { equipmentId }
DELETE /kits/:id/remove-item/:equipmentId
GET    /kits/:id/availability?startDate=&endDate=
       Returns: { status: AVAILABLE|PARTIAL|UNAVAILABLE, missingItems: [] }
```

### Equipment Bookings (Warehouse)
```
GET    /equipment-bookings?inquiryId=
POST   /equipment-bookings
PUT    /equipment-bookings/:id/confirm
PUT    /equipment-bookings/:id/return
POST   /equipment-bookings/bulk-confirm   Body: { ids: [1,2,3] }

GET    /warehouse/check?inquiryId=
       Returns: {
         available: [{ position, kitOrEquipment, serialNo }],
         vendorNeeded: [{ position, equipmentType }],
         conflicts: [{ position, bookedInEvent, dates }]
       }
```

### Vendors
```
GET    /vendors?search=&status=
POST   /vendors
GET    /vendors/:id
PUT    /vendors/:id
GET    /vendors/:id/history
```

### Asset Report
```
GET    /reports/assets/pdf      // Admin only — download PDF
GET    /reports/assets/summary  // JSON summary for dashboard
```
